认证模块

基于 Auth.js v5 的 JWT 策略；支持 Credentials 与 GitHub 两种登录路径；无服务端会话存储。

3.1模块概述

集中式认证入口，密码哈希使用 bcryptjs，salt rounds 默认 10。

3.2文件结构
lib/auth/index.tsNextAuth 实例，导出 auth / handlers / signIn / signOut
middleware.ts基于 auth() 的路由保护
app/api/auth/[...nextauth]/route.tsAuth.js 统一端点
app/api/register/route.ts邮箱注册（独立于 Auth.js）
3.3NextAuth 配置
lib/auth/index.ts
export const { auth, handlers, signIn, signOut } = NextAuth({ adapter: DrizzleAdapter(db, { usersTable, accountsTable, sessionsTable, verificationTokensTable }), providers: [ GitHub({ clientId: env.GITHUB_CLIENT_ID, clientSecret: env.GITHUB_CLIENT_SECRET }), Credentials({ name: "credentials", credentials: { email: { type: "email" }, password: { type: "password" } }, async authorize(credentials) { /* bcrypt.compare + select users */ } }) ], session: { strategy: "jwt" }, callbacks: { session, jwt }, pages: { signIn: "/login" } });

3.6JWT 回调职责

jwt({ token, user }) — 首次登录把 user.id 写入 token.sub。

未登录访问私有页 → 302 /login；已登录访问 /login、/register → 302 /dashboard；API 路由内部再次调用 auth() 做二次校验。

SSE 事件协议

单向流式推送，不使用 EventSource——因为需要 POST body 和自定义 headers。

8.1协议版本

v1，随 /api/generate 部署，无独立版本号。未来破坏性变更应在载荷中加入 version 字段。

8.2连接建立

前端通过 fetch 发起 POST，以 ReadableStream 读取响应体，手动按 \n\n 分帧。

8.3帧格式
data: {"type":"...","...":"..."}\n\n // ⚠️ 不使用 event: 行 —— 所有事件类型通过 JSON 内 type 字段区分
8.4心跳与超时

当前版本不发送心跳。Vercel Serverless 最大持续 60s（Pro 300s），Agent 通常 10–40s 完成。未来如需更长生成：每 15s 发 type: "ping"，前端忽略。

8.5取消机制

前端使用 AbortController.signal 传入 fetch；用户离开或主动取消时 abort()，后端 ReadableStream 的 controller 自动关闭。

8.6事件顺序约束
status (可多次) → agents (恰好 1 次) → agent_start × N → status (并行运行 N 个 Agent...) → 并行：[agent_output | agent_error] × N └ agent_output 紧跟 doc_saved → done (或 error)
8.7前端状态机映射
SSE typeStore 字段更新
statusstreamSteps.push({ label, status: "done" })
agent_startagentToolParts.set({ state: "input-streaming", input })
agent_outputagentToolParts.set({ state: "output-available", output })
agent_erroragentToolParts.set({ state: "output-error", errorText })
doc_savedcompletedDocs.push(label)
errorstatus = "error"; errorMessage = error
done由 reader 循环自然结束时置 completed

前端状态管理

SSE reader 在 Store 闭包中 fire-and-forget 运行，脱离任何组件生命周期。

9.1generation-store · 核心设计

组件订阅 Store 只读取状态；即使组件 unmount，reader 继续消费并更新 Store。

9.2数据结构
stores/generation-store.ts
interface GenerationRun { sessionId: string; sessionTitle: string; status: "running" | "completed" | "error"; streamSteps: StreamStep[]; // 宏观步骤 agentToolParts: Map<string, AgentToolPart>; // 按 agent 键 completedDocs: string[]; // 完成文档标签 errorMessage?: string; abortController: AbortController | null; }
9.3startGeneration 算法
1. 同 sessionId 已有运行 → abort() 旧任务; 2. new AbortController, new GenerationRun → runs.set(); 3. IIFE 异步任务: res = await fetch("/api/generate", { signal }); reader = res.body.getReader(); loop: { done, value } = await reader.read(); buffer += decoder.decode(value, stream: true); lines = buffer.split("\n\n"); buffer = lines.pop(); for (line): JSON.parse → handleSSEEvent; 4. catch: signal.aborted ? 静默 : status = "error";
9.4cancelGeneration

调用 abortController.abort()，从 runs Map 移除该 run。

9.5clearGeneration

不中断（假设已完成或失败），仅从 Map 移除。

9.6workspace-store
interface WorkspaceStore { activeTab: "architecture" | "er" | "requirement" | "api" | "plan"; viewMode: "chat" | "preview" | "split"; setActiveTab(tab); setViewMode(mode); } // 纯 UI 状态，无副作用
